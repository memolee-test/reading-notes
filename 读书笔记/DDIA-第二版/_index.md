# Designing Data-Intensive Applications (第二版)

> **作者**: Martin Kleppmann
> **中文版**: 数据密集型应用系统设计（第二版）
> **主题**: 分布式系统、数据存储、流处理
> **中文翻译来源**: [Vonng/ddia](https://github.com/Vonng/ddia)

## 书籍简介

本书深入探讨了构建数据密集型应用所需的核心概念，涵盖数据库、分布式系统、一致性模型、复制与分区、事务、批处理与流处理等关键主题。

## 核心概念速查

| 概念 | 简述 | 章节 |
|------|------|------|
| 权衡 (Trade-offs) | 数据系统中不存在完美方案，只有取舍 | 第1章 |
| 非功能性需求 (NFRs) | 定义系统应具备的品质属性 | 第2章 |
| 数据模型 (Data Model) | 现实世界到数据结构的映射方式 | 第3章 |
| 存储引擎 (Storage Engine) | 数据在磁盘上的组织与访问方式 | 第4章 |
| 编码与演化 (Encoding) | 数据格式的向后/向前兼容 | 第5章 |
| 复制 (Replication) | 在多个节点间保持数据副本 | 第6章 |
| 分片 (Partitioning/Sharding) | 将大数据集拆分为更小的子集 | 第7章 |
| 事务 (Transactions) | 将多个读写操作视为一个逻辑单元 | 第8章 |
| 共识 (Consensus) | 让所有节点对某个值达成一致 | 第10章 |
| 流式系统哲学 | 流处理超越批处理的思维转变 | 第13章 |
